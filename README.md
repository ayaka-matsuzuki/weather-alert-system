# WeatherAlertSystem
異常気象アラート判定アプリケーション（Java）

## 実行方法
WeatherAlertApplicationのmainメソッドを実行

## 概要
気象庁が公開している降水量データの利用して、
降水量の異常（集中豪雨・干ばつ）を判定する
Java製のドメイン中心アプリケーションです。

本プロジェクトでは、異常気象の定義や判定基準が将来変わることを前提に、
変更に強い設計を重視しています。

## 背景・目的
- 異常気象の判定基準（しきい値・期間）は頻繁に変わる
- データ取得方法や通知方法も将来的に変更される
- そのため、判定ロジックを中心に据えた設計を試みました

## 技術スタック
- Java 17
- Maven
- JUnit 5

## 全体構成（責務分離）
RainfallFetcher（CSVファイルからデータを取得）
↓
Rainfall（降水量ドメイン）
↓
AlertChecker（判定ロジック）
↓
Alert（判定結果）
- データ表現と判定ロジックを分離
- 判定結果はオブジェクトとして扱う設計

## 設計方針
- 判定ロジックはStrategyパターンで分離
- AlertCheckerはインターフェースとして定義
- AlertTypeはenumにより型安全に表現
- ドメイン駆動設計の考え方を取り入れ、
  降水量・期間・アラートといった業務概念を中心に設計
- ドメインオブジェクトはimmutableとし、副作用を排除
- 判定結果はOptionalで表現し、nullを使用しない

## 実装しているアラート
- 集中豪雨アラート（1日降水量ベース）
- 干ばつアラート（一定期間の合計降水量ベース）

※ 現時点では、アラートはオブジェクトとして生成され、
通知や外部出力は行っていません。

## テスト
- 判定ロジックはJUnitによるユニットテストを実装
- 異常あり／なしの両ケースを検証

## 今後の拡張予定
- しきい値の設定ファイル化
- 平年値を基準とした異常判定
- 観測地点別の判定
- Alertの出力（ログ等）の追加

## 今後の追加アラート例
- 長時間強雨アラート（強い雨が同一地点で長時間継続）
- 急変降水アラート（少雨期間後の急激な大雨）
- 累積降水危険アラート（短時間雨量＋期間累積量）
- 平年比異常アラート（平年値からの乖離）
